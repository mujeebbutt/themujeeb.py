Personal Cloud

A lightweight private cloud built with Flask — drag & drop multi-file uploads, 5 GB total cap, progress bar, and a clean gallery with previews for images/videos and cards for docs.

Nextjs Python Tailwind Html
View Code Demo Video

What It Does

  • User accounts with roles and quotas
  • Drag & drop or picker-based multi-file upload (`name="files[]"`).
  • Accepts images, videos, PDFs, and Word docs (.png .jpg .jpeg .gif .bmp .webp · .mp4 .avi .mov .mkv · .pdf .doc .docx).
  • 5 GB total upload cap enforced server-side (`MAX_CONTENT_LENGTH`) with graceful flash messages.
  • Client-side size guard + upload progress bar (XHR) for smooth feedback.
  • Secure filenames via werkzeug.utils.secure_filename; files saved under static/uploads/.
  • Auto gallery: image/video previews; non-preview files show a clean card with “Open”.
  • Error handling: large-file (413) handler and user-friendly flash notices.

Why I Built It

I wanted a simple, private alternative to big clouds for sharing and storing class/work files — with strict size limits, clear progress feedback, and previews — while keeping the codebase tiny and easy to deploy.

Tech Stack

  • Flask + Jinja2 (routing, templating, flash messages)
  • Werkzeug secure_filename (safe file names)
  • Tailwind (CDN) for the glass/dark UI
  • Vanilla JavaScript (drag & drop, XHR upload progress)
  • HTML5 Video for media playback
  • Optional prod: Gunicorn + NGINX (reverse proxy & SSL)

How to Run

  1. Create env & install: python -m venv venv && source venv/bin/activate (Windows: venv\Scripts\activate)
    pip install flask werkzeug
  2. Make upload dir: mkdir -p static/uploads
  3. Run the app: python app.py (or flask run --host 0.0.0.0 --port 5000)
  4. Open: http://localhost:5000 and start uploading
